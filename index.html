Those who want to speed it up, are free to do so. Which is what I did. Like, way up.

I’m glad I stuck with it though, because honestly, this was the most comprehensive explanation of html I’ve come across so far.

I started with cs50, which was mostly c and python. I only got a glimpse of html/css and javascript towards the end. And it was a bit “come on, you’ve done c and python already, this is no big deal” style, which I guess mostly means rushed. I then did The Odin Projects Foundations part, again with a bit html and css (haven’t done the js part yet). It was easy enough, sure. Comprehensive? They probably get to that later.

My key takeaways from the first three classes:

  • Separation of concerns - I loved this way of putting it and the idea of compartmentalizing everything. HTML is for content, CSS is for style and JS is for function.
  • I loved the peanut m&ms analogy too - The peanut is the essence, which is HTML. The chocolate is a bonus, which is CSS and then there’s a thin coating to pimp it up (I personally know the m&ms coating’s true purpose and it sure isn’t an “optional” thing, but I’ll let it slide lol), which is JavaScript. For the sake of accessibility even when the internet is shit, it’s important to focus on the essence and keep it light on the bonus and pimping it up parts.
  • Each tag has its purpose. Its purpose should never be style.
  • The “head” contains the information that is for the browser. The “body” is what is shown to the user. The “you can see my sexy body but you can’t see what’s inside my head” was hilarious and a great way to remember it.
  • Drawing out the elements of a webpage and deconstructing it goes a long way towards understanding what to use where.
  • The tags themselves are easy to lookup and use (mdn is a friend)
